Deployment gates & independence

Engine and Client code deployment path Requires Steam
Source (GitHub) → Build / Validate → Steam Release → Players get update
Server / PA-Net + Infra deployment path No Steam needed
Source (GitHub) → CI / Deploy → Services live → Players benefit immediately

Release little & often

The easier it is to release, the less we need to be perfect — we can always redeploy tomorrow. Our goal is to deploy as frequently as once per day if needed.

  • Ship small changes often instead of storing up work
  • Don't be afraid of mistakes — we can fix forward
  • No waiting around for someone at GA to be available
  • Testing & beta access still available for volunteers
  • Big releases are still possible, but no longer required
