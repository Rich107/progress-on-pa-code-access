Understand the different high-level components

Four Areas. Clean boundaries. Fewer “wait, where does this live?” moments.

Area 1

Source, CI & Deployments

Where code lives and how it ships (for non-engine parts).
Includes
  • Source control
  • CI pipelines & build automation
  • Deployment workflows for services/infrastructure
GitHub
Key idea: This Area supports building & deploying services without requiring a game engine release.
Area 2

Server / PA-Net

Runtime services and online features.
Includes
  • Hosting
  • Matchmaking (MM) & MMR
  • Replays
  • Auth
  • Other backend services
Does not require an engine/Steam release to deploy changes.
Area 3

Game Engine

Game logic, mechanics, and simulation fidelity.
Includes
  • Ammunition mechanics
  • Calculations (damage, ballistics, etc.)
  • Models (tuning / balancing / runtime models)
  • Core game logic & mechanics
Deployment constraint
Steam Release Required
Area 4

Infrastructure

Platform foundations, scale, and connectivity strategy.
Includes
  • AWS foundations
  • Server reduction strategy
  • Lobby
  • Friends list
  • How we move to P2P
AWS
Does not require an engine/Steam release to deploy changes.

Deployment gates & independence

Engine deployment path Requires Steam
Source (GitHub) → Build / Validate → Steam Release → Players get update
Server / PA-Net + Infra deployment path No Steam needed
Source (GitHub) → CI / Deploy → Services live → Players benefit immediately
